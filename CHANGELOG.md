# 更新日志

## [0.3.3] - 2026-05-24

### 新增
- ⚙️ 新增 `contextPrefixLines` / `contextSuffixLines` 设置项，可自定义 FIM 代码补全的上下文参考范围（行数），设为 0 即为整个文件

## [0.3.2] - 2026-05-24

### 优化
- 👁️ 生成提交信息时，SCM 更改栏持续显示状态提示

## [0.3.1] - 2026-05-24

### 新增
- ⚙️ 新增 `enableAutoCompletion` 设置项，默认开启；关闭后可禁用自动补全，仅保留快捷键手动触发

## [0.3.0] - 2026-05-24

### 重大变更
- 🔄 代码补全改用 DeepSeek FIM (Fill In the Middle) API 替代 Chat API
  - 新增 `completionEndpoint` 配置项（默认 `https://api.deepseek.com/beta`），与 Git 提交信息的 API 端点分开设置
  - FIM 补全使用 prompt + suffix 模式，更准确地匹配光标上下文
  - FIM 不支持思考模式，统一使用设置中的模型但不传 thinking 参数
  - 设置中 `apiEndpoint` 现仅用于 Git 提交信息等聊天补全

## [0.2.8] - 2026-05-24

### 修复
- 🐛 修复自动触发代码补全虚影文本不显示的问题（VS Code CancellationToken 超时导致防抖定时器被清除）
- 🎯 状态栏改为旋转图标持久显示，补全等待期间持续可见

## [0.2.7] - 2026-05-24

### 修复
- 🐛 修复手动触发代码补全虚影文本不显示的问题（InlineCompletionItem.range 必须为零长度范围）
- ⚠️ API 密钥未配置时弹出引导提示
- 📋 命令面板中注册 `DeepSeek: 触发代码补全` 命令

## [0.2.6] - 2026-05-24

### 新增
- ⌨️ 新增 `DeepSeek: 触发代码补全` 命令，支持在键盘快捷键设置中自由绑定触发键
- ⚙️ 设置页新增 `triggerKey` 配置项，提供快捷键修改指引入口

## [0.2.5] - 2026-05-24
